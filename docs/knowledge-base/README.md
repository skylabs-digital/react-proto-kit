# 📚 Knowledge Base

This directory contains functional documentation and domain knowledge for the react-proto-kit library.

## Purpose

- Document **design decisions** and their rationale
- Record **known limitations** and workarounds
- Track **consumer patterns** and common use cases
- Capture **lessons learned** from production usage

## Structure

Place new documents here with descriptive names:

```
knowledge-base/
├── README.md                          # This file
├── error-handling-patterns.md         # How consumers handle errors
├── url-state-management.md            # URL-driven UI patterns
└── ...
```

## Conventions

- Use Markdown with clear headings
- Include code examples when relevant
- Date your entries for freshness tracking
- Link to related RFCs in `docs/rfcs/` when applicable
